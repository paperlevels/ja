# Paperlevels テスト仕様書

> 対象環境: ローカル Supabase（自由な CRUD 操作が可能な状態）

---

## 1. 導入パッケージ

```bash
npm install -D vitest @vitest/ui playwright @playwright/test jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event
```

---

## 2. テスト構成

| レイヤー | テスト種別 | ツール | 対象 |
|---------|-----------|--------|------|
| 単体 | Unit Test | Vitest | バリデーション、ユーティリティ、型 |
| API | Integration Test | Vitest + 実DB | API Routes (`/api/*`) |
| DB/RLS | Integration Test | Vitest + 実DB | RLSポリシー、トリガー |
| E2E | End-to-End | Playwright | ブラウザ操作フロー |

---

## 3. API 統合テスト (`tests/api/`)

### `loglines.test.ts`

| # | ケース名 | 検証内容 |
|---|---------|---------|
| 1 | 正常投稿 | 140文字以内のログラインがDBに保存され、`id`（=`content` と同じ値）が返る |
| 2 | 空文字投稿 | 400エラー、DBにレコードが増えない |
| 3 | 空白のみ投稿 | 400エラー、トリム後空文字として拒否される |
| 4 | 141文字投稿 | 400エラー、バリデーション拒否 |
| 5 | 禁止文字投稿 | `/ ? & # % \` など含む場合、400エラーで拒否される |
| 6 | 重複投稿 | 同じ内容が既に存在する場合、409エラーで拒否される |

### `comments.test.ts`

| # | ケース名 | 検証内容 |
|---|---------|---------|
| 7 | 正常コメント | コメント保存 + `loglines.comment_count` が+1される |
| 8 | 空コメント | 400エラー |
| 9 | 5001文字コメント | 400エラー |
| 10 | `loglineId` 未指定 | 400エラー |
| 11 | 存在しないloglineId | 500エラー（FK制約違反） |

### `share.test.ts`

| # | ケース名 | 検証内容 |
|---|---------|---------|
| 12 | 正常シェアカウント | `share_count` が+1される |
| 13 | `id` 未指定 | 400エラー |
| 14 | 存在しないID | 500エラー |

### `admin.test.ts`

| # | ケース名 | 検証内容 |
|---|---------|---------|
| 15 | 未認証で削除 | 401エラー、レコードが残る |
| 16 | 未認証でカテゴリ更新 | 401エラー |
| 17 | 認証後の削除 | 200、レコードが消える |
| 18 | 認証後のコメント削除 | `loglineId` を指定して削除、`comment_count` が-1される（0未満にならない） |
| 19 | 認証後のカテゴリ更新 | `category` が更新される |
| 20 | 空カテゴリ更新 | `category` が空/空白の場合、`null` で更新される |

## 7. 実装優先度

1. **API統合テスト**（`tests/api/`）— 最優先
2. **DB/RLSテスト**（`tests/db/`）— セキュリティ担保のため優先
3. **データ層テスト**（`tests/lib/`）— 並行して実装可能
4. **E2Eテスト**（`e2e/`）— UIフロー確定後に実装
